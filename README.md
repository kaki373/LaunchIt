# LaunchIt

Start.bat 起動型の WebUI 系アプリ(ComfyUI / Forge など)を、トレイ常駐 +
ホットキーのポップアップから **起動 / 再起動 / 停止** できる Windows 用の
軽量ランチャー。フォルダや URL、DCC ツール(UE / 3ds Max / After Effects
など)の起動、実行中の Claude Code / Codex セッションの検出と前面化にも
対応しています。

- Python 3.10+ / tkinter(標準)+ pystray + Pillow だけで動く単一スクリプト
- メモリ約 65MB、ポーリングは軽い TCP 接続とウィンドウ列挙のみ

## セットアップ

```
pip install pystray pillow
python launchit.py        # 初回起動で launchit.json が生成される
```

- 常用は `LaunchIt.vbs`(コンソールなしで常駐。pythonw が PATH にあること)
- ログオン時に自動起動: `register_startup.bat` を一度実行
- 2 回目に起動しようとすると、既存インスタンスのポップアップが開くだけです
  (多重起動しません)

## 使い方

| 操作 | キー / マウス |
|---|---|
| ポップアップ表示 / 非表示 | `Ctrl+Space`(変更可)、またはトレイアイコンをクリック |
| 絞り込み | そのまま文字入力(部分一致 + ゆるい順序一致) |
| 項目の追加 | 検索欄右の **[+] ボタン** → フォルダ / アプリ / URL |
| 項目の編集 | [+] メニューかトレイの「項目を編集」→ 並び替え・名前変更・追加・削除 |
| 起動 / 開く | `Enter` かダブルクリック |
| 実行中の項目で `Enter` | **既存ウィンドウを手前に表示**(見つからなければ `url` を開く) |
| 再起動 | `Ctrl+R`、または右クリック → 再起動 |
| 停止(プロセスツリーごと) | `Ctrl+D`、または右クリック → 停止 |
| その項目のフォルダを開く | `Ctrl+Enter` |
| 閉じる | `Esc`(フォーカスが外れても自動で閉じる) |

リストの表示: `●` 実行中(緑)/ `○` 停止中 / `▶` 状態を追跡しない単発起動 /
`⌨` 検出された CLI セッション / `📁` フォルダ / `🌐` URL。

ポップアップは**マウスカーソルがあるモニタ**の右下(タスクバーのすぐ上)に
出ます。列数・横幅・高さ・配置は編集ウィンドウ上部のレイアウト欄で変更でき、
編集中は実物のポップアップがライブプレビューされます。

## 設定 (launchit.json)

初回起動時に自動生成されます。保存すれば次のポップアップ表示時に自動反映
(ホットキーの変更も再起動不要)。

```jsonc
{
  "hotkey": "ctrl+space",                // ctrl/alt/shift/win + 英数字・F1-F12・space など
  "hotkey_passthrough": ["AfterFX.exe"], // このexeが前面の間はホットキーを素通しする
  "position": "bottom-right",            // bottom-right / center
  "columns": 2,                          // 列数
  "width": 720,                          // 横幅(px)
  "height": 0,                           // 高さ(px)。0 = 項目数に合わせて自動
  "items": [
    {
      "name": "My WebUI app",            // 表示名(検索対象)
      "type": "app",                     // app / folder / url / wingroup
      "path": "C:/path/to/start.bat",    // .bat / .cmd / .exe / .py
      "url": "http://127.0.0.1:8188",    // 任意: 稼働判定ポート + Enter で開くURL
      "stop_path": "C:/path/to/stop.bat",// 任意: 停止スクリプト(なければ taskkill /T)
      "cwd": "C:/path/to",               // 任意: 作業フォルダ(既定は path の場所)
      "args": ["--port", "8189"],        // 任意: 追加引数
      "track": false,                    // 任意: false で状態追跡なしの単発起動
      "color": "#e8926b"                 // 任意: 表示色
    },
    { "name": "Home", "type": "folder", "path": "C:/Users/you" },
    { "name": "GitHub", "type": "url", "url": "https://github.com" },
    { "name": "Claude", "type": "wingroup", "title_re": "^[✳⠀-⣿]", "color": "#e8926b" }
  ]
}
```

## 実行中の判定(2系統)

1. **プロセス追跡** — LaunchIt が起動したプロセス。さらに、外で起動済みの
   アプリも取り込みます(.bat はコマンドライン照合、.exe は実行ファイルの
   パス照合。起動時・トレイの「再スキャン」・ポップアップを開いたとき)
2. **ポート稼働チェック** — ローカル `url` か `check_port` を持つ項目は、
   ポートへの接続で判定。「start.bat が本体を別ウィンドウで起動して自分は
   終了する」タイプのアプリでも正しく ● になります。ポーリングは表示中
   5 秒 / 非表示中 20 秒間隔

ポート検出のみ(プロセス未追跡)のアプリの停止・再起動には `stop_path` が
必要です。自動再起動型の監視プロセス(例: `concurrently --restart-tries -1`)
を持つアプリは、監視役ごと止める停止スクリプトを用意してください
(`stop_ai_toolkit.py` が実例: ポートの所有プロセスから親をたどって
ツリーごと終了します)。

## CLI セッション検出 (wingroup)

`"type": "wingroup"` の項目は、タイトルが `title_re` に一致する**ターミナル
ウィンドウ**を列挙し、1 セッション = 1 項目として表示します。Enter で前面化。

- Claude Code はタブタイトル先頭の「✳ / スピナー」で検出できます
- タイトルを設定しない CLI(Codex など)は、起動ラッパーに
  `title Codex - %CD%` のような 1 行を足すと検出できるようになります
- Windows Terminal はアクティブなタブのタイトルしか見えないため、背後の
  タブのセッションは検出できません(別ウィンドウなら全部見えます)
- 一覧は表示中毎秒更新。一度検出したセッションはタイトルが一時的に変わっても
  (ウィンドウが生きている限り)2 分間はリストに残ります

## 手前に表示の探し方

- 追跡中のツール(UE / 3ds Max など)は**自プロセスツリー**のウィンドウを
  特定します(最小化からも復元)
- WebUI 系はウィンドウタイトルに項目名を含む**ブラウザウィンドウ**
  (Firefox 系 / Chromium 系のウィンドウクラスで判定)を探し、見つからなければ
  新しいタブで URL を開きます。非アクティブなタブの直接アクティブ化は
  ブラウザ拡張なしでは不可能です

## 注意

- グローバルホットキーは他ツールと競合します(例: PowerToys Peek の
  Ctrl+Space)。どちらかの割り当てを変えてください
- 「停止」は taskkill による強制終了です。未保存の作業があるツールには
  使わないでください
- LaunchIt を終了・再起動しても、起動済みのアプリは動き続けます
